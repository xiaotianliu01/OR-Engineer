# Data Interface Notes

CP data should define:

- entities, tasks, jobs, resources, or locations,
- domains for every decision variable,
- time granularity and calendars,
- hard constraints and soft preferences,
- objective weights and units,
- optionality and eligibility,
- symmetry or equivalence classes,
- validation rules for extracted solutions.

Keep task data outside the reusable asset and record data paths in `run_records/run_config.json`.

# Schedule Modeling Notes

When modeling schedules, record:

- task set,
- resource set,
- eligible resources,
- durations,
- release dates,
- due dates,
- precedence relations,
- setup times,
- calendars and downtime,
- optional tasks,
- hard and soft constraints,
- objective terms and units.

Validate extracted schedules:

- resource conflicts,
- precedence violations,
- time-window violations,
- missing or duplicate assignments,
- objective reconstruction.
